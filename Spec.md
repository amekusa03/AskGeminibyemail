# GASによるGeminiメール自動返信システムの構築

## 1. システム概要と仕様

本レポートでは、Google Apps Script（GAS）を用い、GmailをインターフェースとしてGemini API（Google Generative AI API）との対話を実現するシステムの構築手順を詳述する。本システムは、ブラウザや専用アプリの利用が制限された環境下においても、自分宛のメール送受信のみで最新のAIモデルと対話することを目的としている。

なお、本システムはGoogle AI Studioで発行したAPIキーを利用するため、複雑なGoogle Cloud Projectのセットアップは不要である。

### 主要な仕様と動作原理

* 技術スタック: Google Apps Script (GmailApp, UrlFetchApp)
* 実行形態: 5分間隔などの時間主導型トリガーによる「ポーリング方式」（定期チェック型）
* トリガー条件: メールの件名に特定のキーワード「AskGemini」が含まれていること

### 動作フロー

1. 検索: 受信トレイから件名に「AskGemini」を含む未読メールを検索する。
2. 本文抽出: 該当スレッドの最新メッセージから本文（ユーザーの質問）を抽出する。
3. API送信: 抽出したテキストをプロンプトとしてGemini APIに送信する。
4. 返信: APIからの生成レスポンスを、元のメールスレッドに返信する。
5. 既読処理: 二重処理（重複返信）を防止するため、処理済みのスレッドを既読にする。

## 2. 事前準備：Gemini APIキーの取得

Gemini APIを利用するためには、認証用となるAPIキーが必要である。

1. Google AI Studio にアクセスし、Googleアカウントでログインする。
2. サイドメニューの「Get API key」をクリックする。
3. 「Create API key in new project」を選択し、生成された文字列（APIキー）を安全にコピーしておく。

## 3. 実装コード

本システムの実装コードは、`checkAndReplyWithGemini.js` ファイルに記述されています。このドキュメントでは、そのコードを前提とした設定手順と運用方法を説明します。

## 4. Google Apps Script での設定手順

### 4.1 プロジェクトの作成とコードの貼り付け

1. Google Apps Script エディタを開き、「新しいプロジェクト」を作成する。
2. デフォルトの myFunction を削除し、上記「3. 完全版実装コード」をすべて貼り付けて保存する。

### 4.2 スクリプトプロパティの設定

APIキーをコード内にハードコードせず、安全に管理するための設定を行う。

1. エディタ左サイドバーの「プロジェクトの設定（歯車アイコン）」をクリックする。
2. ページ下部の「スクリプト プロパティ」セクションにある「スクリプト プロパティを編集」をクリックする。
3. **「+ スクリプト プロパティを追加」** をクリックし、以下の通り入力する。
   * プロパティ: GEMINI_API_KEY
   * 値: Google AI Studioで取得したAPIキー

4. 「スクリプト プロパティを保存」をクリックする。

### 4.3 サービスの有効化と権限承認

初回実行時は、Googleアカウントによるアクセス権限の承認が必要となる。

1. エディタ上部の関数選択で checkAndReplyWithGemini を選択し、「実行」をクリックする。
2. 「承認が必要」ダイアログで「権限を確認」をクリックし、自分のアカウントを選択する。
3. 重要（警告画面）: 「Google hasn't verified this app（このアプリは確認されていません）」という警告が表示された場合、左下の**「Advanced（詳細）」をクリックし、さらに一番下の「Go to [プロジェクト名] (unsafe)」**をクリックして進む必要がある。
4. アクセスリクエストを許可すれば、初回実行が完了する。

## 5. トリガーの設定（自動化）

システムを全自動で稼働させるため、定期的な実行を予約する。

1. エディタ左サイドバーの「トリガー（時計アイコン）」をクリックする。
2. 右下の「トリガーを追加」ボタンをクリックする。
3. 設定を以下のように調整し、保存する：
   * 実行する関数: checkAndReplyWithGemini
   * イベントのソース: 「時間主導型」
   * 時間ベースの型: 「分ベースのタイマー」
   * 時間の間隔: 「5分ごと」

## 6. 使用方法と運用上のアドバイス

### 基本的な利用手順

* メール送信: 自分宛（スクリプトを設置したGoogleアカウント）に、件名に AskGemini を含めたメールを送信する。本文に質問を記述する。
* 動作確認: トリガー設定に基づき、最大5分以内にGeminiからの回答がスレッド形式で返信される。
* 即時テスト: 5分待てない場合は、GASエディタの「実行」ボタンを手動で押すことで即座に処理が開始される。

### トラブルシューティング

* 返信が届かない場合: エディタ左メニューの「実行数」をクリックし、ログを確認する。APIステータスコードが400や429（レート制限）になっていないかを確認。
* 二重返信の防止: 本システムは thread.markRead()（既読にする）をトリガーの条件としている。手動でメールを未読に戻すと、再実行時に再度Geminiが反応するため注意が必要である。
* 本文抽出の精度: メールの署名や過去の引用返信が含まれている場合、それらすべてがGeminiへのプロンプトとして送信される。精度の高い回答を得るには、質問内容のみを本文に記載することが推奨される。
